# Agent Localization 双模式修缮记录（20260316-002921）

本次修缮按已确认计划实现了 `agent/run.py` 的 localization 双模式（`pipeline|lite`）与单一 CLI 枚举切换，并保持 `LAB_LOCALIZE_OUTPUT` 预计算结果优先的既有行为。

## 1. 需求与目标

- 单一枚举参数：`--localize-mode {pipeline,lite}`。
- 默认模式：`pipeline`（完全对齐 `pipeline/localize.py` 方法）。
- 兼容策略：先用 `LAB_LOCALIZE_OUTPUT`；仅在当前 issue 缺失预测时，执行 runtime localization。
- 同步文档：`agent/.env.example`、`agent/README.md`、`agent/README-zh.md`。

## 2. 实施过程（按步骤）

1. 阅读并核对当前 `agent/run.py` localization 路径、CLI 参数入口、`build_task_description` 注入逻辑。
2. 新增 localization 模式常量与解析逻辑（CLI > ENV > 默认）。
3. 在 `run.py` 中新增 pipeline 对齐配置项（issue desc/debug 相关环境变量）。
4. 将 runtime localization 拆分为：
   - `_run_runtime_localization_lite(...)`
   - `_run_runtime_localization_pipeline(...)`
   - `_run_runtime_localization(...)` dispatcher
5. 为 `pipeline` 模式补齐 pipeline 风格流程：
   - 首个 missed test case 提取
   - pipeline 风格 issue 描述
   - `opt -debug` 上下文采集（含 build-before-debug 与 timeout）
   - file/function prompt 结构对齐
6. 将模式参数贯通到主流程：`main -> fix_issue -> build_task_description -> runtime localize`。
7. 更新 `.env.example` 与双语 README，补充模式配置与命令示例。
8. 执行语法与入口验证命令，确认改动可运行。
